# 쇼츠 추출 규칙 (설교 본문 → 후보 5개 → 3개 선택)

설교 본문 자막을 분석해, 요약·키워드를 먼저 만들고 쇼츠 **후보 5개**를 뽑는다.
그중 가장 좋은 **3개를 추천 선택**으로 표시한다(최종 선택은 사용자가 한다).
시각은 반드시 자막 타임스탬프를 그대로 사용하고, 임의로 추정하지 않는다.

## 입력 범위
- 분석 대상은 **설교 본문 구간**(설교규칙의 `sermon_start` ~ `sermon_end`)뿐이다.
- 여는 기도 이전(광고·공지·봉독 전), 그리고 **축도 구간(benediction)은 제외**한다.

## 1단계: 요약과 키워드
- summary: 이 설교의 핵심 메시지를 3~4줄로 요약.
- keywords: 설교에서 **반복하거나 강조해서** 말하는 핵심 단어 5~8개.

## 2단계: 쇼츠 후보 5개

각 후보는 아래 세 가지 성격 중 하나를 가진다. 5개는 **세 성격을 모두 포함**하도록 구성한다.
(권장 분배: 후킹형 2 · 예화형 2 · 핵심선언형 1, 소재가 부족하면 분배는 조정 가능)

### ① 후킹형
- 도입의 질문, 충격적 한마디, 호기심을 끄는 대목.
- 시작 3초가 강해야 한다(영상 앞이 약하면 사람들이 바로 넘김).

### ② 예화형
- 이야기·사례·비유 부분. 그 자체로 **완결된 한 토막**이어야 한다.
- 몰입감이 있고, 끝에 작은 반전이나 여운이 있으면 좋다.

### ③ 핵심선언형
- 메시지가 압축된 결정적 문장. summary·keywords와 **가장 강하게 연결**되는 부분.

## 3단계: 3개 추천 선택
- 후보 5개 중 점수(score)가 높고 성격이 서로 다른 **3개**를 골라
  `selected_candidate_ids` 에 후보 id 로 담는다.
- 가능하면 세 성격(후킹형·예화형·핵심선언형)이 골고루 들어가게 추천한다.

## 공통 규칙 (매우 중요)
- 길이: 각 쇼츠는 **15~30초**(end - start)를 기본으로 한다.
  문장 완결성이 우선이며, 부득이하면 **최대 40초**까지 허용한다.
- 경계: 문장이 **자연스럽게 시작하고 끝나는** 지점으로 자른다. 말이 중간에 잘리면 안 된다.
- 그 구간만 따로 봐도 **말이 되는** 토막이어야 한다.
- 후보 5개는 서로 다른 구간(시간이 겹치지 않게).

## 각 후보에 함께 만들 정보
- id: 후보 식별자("cand_001" 형식, 001부터 순번).
- type: "후킹형" / "예화형" / "핵심선언형".
- title: 호기심을 끄는 추천 제목 한 줄.
- hook_line: 영상 맨 앞 3초에 띄울 문장(자막에서 가져오거나 압축).
- highlight: 화면에 **크게 강조**할 핵심 한 문장(자막에 있는 표현 그대로).
- hashtags: 3~5개(예: #설교 #믿음 #은혜).
- reason: 왜 이 구간인지(키워드/요약과의 연결 근거 한 줄).
- score: 쇼츠로서의 추천 점수(0~100 정수).

## 출력 형식 (반드시 JSON만, 다른 말 없이)
```json
{
  "summary": "이 설교의 핵심 3~4줄 요약",
  "keywords": ["은혜", "기다림", "돌아옴"],
  "candidates": [
    {
      "id": "cand_001",
      "type": "후킹형",
      "start": "00:31:20",
      "end": "00:31:46",
      "duration_sec": 26,
      "title": "당신이 끝이라고 생각한 그 순간",
      "hook_line": "정말 끝이라고 생각한 적 있습니까?",
      "highlight": "끝이 아니라 시작입니다",
      "hashtags": ["#설교", "#은혜", "#믿음"],
      "reason": "도입 질문 + 키워드 '시작'과 연결",
      "score": 88
    },
    { "id": "cand_002", "type": "예화형", "...": "..." },
    { "id": "cand_003", "type": "예화형", "...": "..." },
    { "id": "cand_004", "type": "후킹형", "...": "..." },
    { "id": "cand_005", "type": "핵심선언형", "...": "..." }
  ],
  "selected_candidate_ids": ["cand_001", "cand_003", "cand_005"]
}
```
- start, end 는 "HH:MM:SS", duration_sec 는 15~40(기본 15~30).
  (앱 엔진이 `to_seconds` 로 초 단위 float 로 변환해 `shorts_candidates.json` 에 저장한다.)
- candidates 는 정확히 5개, id 는 cand_001 부터 순번.
- selected_candidate_ids 는 후보 중 정확히 3개의 id.
- 시각은 자막 타임스탬프에서 그대로 가져온다(추정 금지).
