# 설교 시작·끝 판별 규칙 (믿음의교회 주일 예배)

이 문서는 AI가 자막을 보고 설교의 **시작 시각**과 **끝 시각**을 찾을 때 따르는 규칙이다.
시각은 반드시 자막에 표시된 타임스탬프를 그대로 사용하고, 임의로 추정하지 않는다.

## 부 구분
- 영상 제목에 "주일 1부 / 2부 / 3부"가 들어 있으면 그 부의 규칙을 적용하고, `part` 에 "1부"/"2부"/"3부" 를 넣는다.
- 제목에 "중고등부"(또는 "중·고등부", "학생부")가 들어 있으면 중고등부 규칙을 적용하고, `part` 에 "중고등부" 를 넣는다.
- 제목에 부 표시가 없으면(수요·금요예배 등) 아래 흐름 신호로 판단하고, `part` 에 제목에서 추정한 예배명(예: "수요예배", "금요예배")을 넣되 알 수 없으면 "미상" 으로 둔다.

## 시작점 규칙

### 1부
- 흐름: (성경봉독) → 설교 시작
- 시작점 = **성경봉독/본문 멘트가 처음 나오는 자막의 시각**
  (예: "요한복음 6장", "오늘 본문은…", "함께 읽겠습니다", "봉독하겠습니다")

### 2부
- 흐름: 성가대 찬양 → "아멘" → 짧은 기도 → 메시지 시작
- 시작점 = 여는 짧은 기도("…이름으로 기도합니다 아멘") **직후**, 보통 본문 멘트가 나오는 지점

### 3부
- 성가대가 있는 날: 성가대 찬양 → "아멘" → 짧은 기도 → 설교
  - 시작점 = 여는 기도 직후
- 성가대가 없는 날: 대표기도자 "…이름으로 기도합니다 아멘" → 성경봉독 → 설교
  - 시작점 = 그 기도 직후의 **성경봉독 멘트**
- 판별: 여는 기도 직후 가까이에 성경봉독 멘트가 있으면 "성가대 없는 날"로 본다.

### 중고등부
- 흐름: 대표기도 → 사회자 **성경 암송** → (암송 끝) → 설교자 등장 → 설교 시작
- 시작점 = **성경 암송이 끝난 직후**, 설교자가 말을 시작하는 지점.
- 성경 암송 신호: 사회자가 성경 구절을 외우거나 다 함께 암송하는 대목(같은 구절을 또박또박 읽듯 말함, "다 함께 암송", "○○서 ○장 ○절" 등).
- 주의: 암송 자체는 설교가 아니다. **암송이 끝나고 설교자(다른 목소리)가 본론을 시작하는 지점**을 시작으로 잡는다.
- 끝점은 아래 "끝점 규칙(공통)"과 같다: 설교 후 마무리 기도 → 그 직후 찬송가가 이어지는 기도가 끝.

## 끝점 규칙 (모든 부·중고등부 공통)
- 설교는 설교자의 **마무리 기도**로 끝나고, 그 기도가 끝나면 **곧바로 찬송가(찬송/찬양)**가 이어진다.
- 끝점 = 이 **마무리 기도가 끝나는 시각** (= 찬송가가 시작되기 직전).
  - 대표 멘트: "예수 그리스도 이름으로 기도합니다", "주 예수님 이름으로 기도합니다", "예수 그리스도의 이름으로 기도드립니다"
  - 변형 허용: 띄어쓰기 차이("기도 합니다"), 두 줄로 쪼개진 경우 이어서 판단.

### 진짜 끝 기도를 가려내는 법 (중요)
- 영상에는 같은 기도 문구가 여러 번 나올 수 있다(여는 기도, 중간 기도 등).
  **그 직후에 찬송가가 이어지는** 기도가 설교를 닫는 진짜 마무리 기도다.
- 따라서 후반부에서 "…이름으로 기도합니다 (아멘)" **다음에 찬송/찬양/찬송가** 관련 자막이
  나오는 지점을 찾고, 그 **기도의 마지막 자막 시각**을 끝점으로 한다.
- 찬송 시작 신호 예: "찬송", "찬양", "○○○장", "다 같이 일어나", "함께 부르겠습니다", "찬송가".

### 못 찾을 때
- 마무리 기도 + 찬송 패턴을 못 찾으면, 후반부에서 마지막으로 나오는
  "…이름으로 기도합니다"의 시각을 끝점으로 한다.
- 그것도 없으면 후반부 마지막 "아멘" 또는 축원("축복합니다/축원합니다")을 끝점으로 본다.

## 축도 구간 규칙 (설교 끝 이후)
예배 흐름: 설교 마무리 기도 → **찬송가** → **축도**(목사님 축복 기도) → 예배 종료

- **축도 시작점** = 설교 끝(마무리 기도) 다음에 부른 **찬송가가 끝난 뒤**,
  목사님이 다시 **축복 기도를 시작하는 시각**.
  - 즉 "설교 끝 → 찬송 → (찬송 종료) → 목사님 음성 다시 시작" 의 그 지점.
- **축도 끝점** = 축도의 마지막 멘트 시각.
  - 대표 멘트: "지금부터 영원토록 함께 할지어다 아멘"
  - 변형 허용: "영원토록 함께 있을지어다", "너희와 함께 있을지어다",
    "지금부터 영원토록…", 띄어쓰기/표기 차이("할찌어다"), 두 줄로 쪼개진 경우 이어서 판단.
- 축도는 설교 마무리 기도와 **다르다**. 축도는 찬송가 **뒤**에 오고,
  "영원토록 함께 할지어다" 류의 축복 선언으로 끝난다.
- 축도 구간을 찾지 못하면 benediction_start, benediction_end 를 null 로 둔다.


## 자막 줄 경계 기준
- 자막 한 줄은 보통 2~4초 구간(줄의 start~end)을 가진다. 경계 시각은 다음처럼 잡는다.
  - **시작점**(sermon_start, benediction_start) = 해당 멘트가 나오는 자막 줄의 **start** 시각.
  - **끝점**(sermon_end, benediction_end) = 해당 멘트가 끝나는 자막 줄의 **end** 시각.

## 주의
- 여는 기도와 맺는 기도는 **같은 문구**("…이름으로 기도합니다")를 쓴다.
  앞부분에 나오면 시작 경계, 뒷부분에 나오면 끝으로 구분한다.
- 광고·공지·교독문은 설교가 아니다. 본문 봉독 또는 여는 기도 이후가 설교다.

## 출력 형식 (반드시 JSON만, 다른 말 없이)
```json
{
  "part": "2부",
  "sermon_start": "00:25:36",
  "sermon_end": "01:15:07",
  "start_reason": "성경봉독 멘트: 오늘 본문은 …",
  "end_reason": "예수 그리스도 이름으로 기도합니다 (직후 찬송가 이어짐)",
  "benediction_start": "01:18:10",
  "benediction_end": "01:20:05",
  "benediction_start_reason": "찬송가 종료 후 목사님 축복 기도 시작",
  "benediction_end_reason": "지금부터 영원토록 함께 할지어다 아멘"
}
```
- sermon_start, sermon_end, benediction_start, benediction_end 는 "HH:MM:SS" 형식.
  (앱 엔진이 `to_seconds` 로 초 단위 float 로 변환해 `sermon_analysis.json` 에 저장한다.)
- 축도 구간을 못 찾으면 benediction_start, benediction_end 는 null.
- 시각은 자막 타임스탬프에서 그대로 가져온다(추정 금지).
